go-vikunja/vikunja#2680 — Multiple Labels on a task are ignored — v2.3.0

BUG CONFIRMED
When creating a task via Vikunja's Quick-Add syntax with multiple new labels (e.g. My task *label1 *label2), only the first label is persisted. The second concurrent PUT /api/v1/tasks/{id}/labels call fails with HTTP 500 (race condition/transaction conflict on the label_tasks table in SQLite). The UI reflects this by NOT clearing the input field — matching the reporter's description that "the task remains in the Add a task field".
