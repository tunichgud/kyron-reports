Bug #2672 — Kanban Column Count Wrong in Saved Filters

What goes wrong?

When a saved filter's Kanban view has a bucket limit set, adding the first task to that bucket fails with "exceeded limit" — even though the bucket is empty (count=0, limit=1). The UI correctly displays 0/1, but the backend rejects the add operation.

In contrast, a regular (non-filter) project bucket with identical limit=1, count=0 accepts the task without error.

Affected version: v2.3.0

Root cause: The bucket limit check in the backend uses an incorrect task count for filter-project buckets. It counts tasks globally or against the wrong scope, making an empty filter-view bucket appear full.
