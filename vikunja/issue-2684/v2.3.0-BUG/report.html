BUG CONFIRMED on v2.3.0

What goes wrong: Vikunja's keyboard shortcut handler in frontend/src/helpers/shortcut.ts uses event.code (physical key position, QWERTY-named) instead of event.key (the actual character the user's layout produces). This means shortcuts only work for QWERTY users. A Dvorak user pressing their "k" keycap (which sits at the QWERTY "V" position) produces event.code="KeyV", but Vikunja checks for event.code="KeyK" — mismatch, shortcut never fires.
