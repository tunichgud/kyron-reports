Technische Begruendung

Architektonisch

Vikunja 2.3.0 wechselte den Desktop-Login von internem Browser-Dialog auf 'System-Default-Browser oeffnen'. Der Bug liegt im Uebergang Electron-App -> externer Browser -> Callback. Ein Playwright-Browser kann weder den Electron-Start noch den Handoff zwischen zwei realen Browsern simulieren.

Semantisch

Der Auth-Code ist ein Single-Use-OAuth-Artefakt (RFC 6749 Section 10.5). Er wird nach dem Token-Austausch korrekt aus der URL entfernt. Das ist kein Defekt, sondern Sicherheitsdesign. Ein Test, der dieses korrekte Verhalten als Fehler assertet, waere falsch positiv.

Scope

Kyron-Scope (Projektmemo): 'nur UI-Bugs (Playwright-tauglich), keine Backend-only Issues'. Desktop-App + Multi-Browser-Handoff faellt ausserhalb dieses Scopes.
