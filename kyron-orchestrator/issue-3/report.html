kyron-orchestrator Issue #3: security: /actuator/health on port 9091 requires OAuth2 login BUG CONFIRMED

Version: main | Job: 0bc10200-1fd8-4bd6-a3cd-fba963bb2d9c | Reproduced: 2026-04-24 | Issue: https://github.com/tunichgud/vlyby-kyron-orchestrator/issues/3

BUG CONFIRMED

GET http://127.0.0.1:9091/actuator/health returns HTTP 302 redirecting to /oauth2/authorization/google instead of HTTP 200 {"status":"UP"}. The management health endpoint is not publicly accessible because SecurityConfig.webFilterChain applies to all ports and /actuator/health is not in the permitAll() list.

Root Cause

SecurityConfig.webFilterChain is not restricted to port 8080. It covers all ports including the management port 9091. The /actuator/health path is absent from the permitAll() matcher at SecurityConfig.java:55:

// SecurityConfig.java:55
.requestMatchers("/api/version", "/error", "/login/**", "/oauth2/**").permitAll()
// ← /actuator/health not listed

Expected: GET /actuator/health on port 9091 returns HTTP 200 {"status":"UP"} without authentication.

Actual: GET /actuator/health on port 9091 returns HTTP 302 with Location: http://127.0.0.1:9091/oauth2/authorization/google.

Suggested Fix

Add /actuator/health to permitAll():

.requestMatchers("/api/version", "/actuator/health", "/error", "/login/**", "/oauth2/**").permitAll()

Alternatively, restrict webFilterChain to port 8080 only using ServerPortRequestMatcher and add a dedicated permit-all chain for port 9091.
