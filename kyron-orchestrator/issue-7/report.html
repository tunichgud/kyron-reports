kyron-orchestrator Issue #7: updatePhase reports "Phase not found" when job does not exist BUG CONFIRMED

Version: main | Job: 8cc06df7-3e43-44eb-990e-1ab665cbfd0d | Reproduced: 2026-04-24 | Issue: https://github.com/tunichgud/vlyby-kyron-orchestrator/issues/7

BUG CONFIRMED

Calling kyron_phase_update with a non-existent job UUID returns Error: Phase not found: triage for job <id> instead of Error: Job not found: <id>. The misleading message implies the job exists but lacks the named phase, making it impossible to distinguish a wrong job ID from a wrong phase name.

Root Cause

JobService.updatePhase at line 177 queries job_phases without first checking job existence:

public void updatePhase(UUID jobId, PhaseUpdateRequest request) {
    JobPhase phase = jobPhaseRepository.findByJobIdAndPhaseName(jobId, request.phase())
        .orElseThrow(() -> new IllegalArgumentException(
            "Phase not found: " + request.phase() + " for job " + jobId));
    // ← no prior check: jobRepository.existsById(jobId)
}

When the job doesn't exist, no phases are found either, so the code falls into the same orElseThrow branch as a genuinely missing phase name.

Suggested Fix

public void updatePhase(UUID jobId, PhaseUpdateRequest request) {
    if (!jobRepository.existsById(jobId)) {
        throw new IllegalArgumentException("Job not found: " + jobId);
    }
    JobPhase phase = jobPhaseRepository.findByJobIdAndPhaseName(jobId, request.phase())
        .orElseThrow(() -> new IllegalArgumentException(
            "Phase not found: " + request.phase() + " for job " + jobId));
    ...
}
