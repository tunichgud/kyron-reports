Reusable Workflow Cached

Bug: When a main workflow reuses a workflow from another repository, the reusable workflow file is cached on the runner. Modifications to the reusable workflow are not reflected on re-run.

Version: latest (Gitea)
Test Type: API-based automated regression test
Root Cause: Runner caches reusable workflows without checking for updates when ref is a branch (@master)
